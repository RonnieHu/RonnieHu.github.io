Git与版本控制

工具 2018/10/13

Linux之父Linus Torvalds除开源Linux操作系统外，还为编程界贡献了另一个优秀的工具，Git。Git是分布式版本控制工具，用于管理代码的分支与版本迭代，其与代码托管平台Github配合使用是当代绝大多数互联网公司的共同选择。

Git的前世今生

Linus在开源Linux操作系统之后，世界上各地的程序员纷纷踊跃加入Linux的开发与维护的行列中，此时代码的管理就成为一个亟待解决的问题。Linus大神首先选择了一个商业的版本控制工具Bitkeeper，Bitkeeper的母公司Bitmover出于开源精神授权Linux社区免费使用这个版本控制工具，此时开发Samba的一名程序员试图破解Bitkeeper的协议，但是被Bitmover公司发现，Bitmover认为这侵犯了自己的利益，表示要收回Linux社区的免费使用权。

于是Linus花了两周时间自己用C语言写了一个分布式版本控制工具，这就是Git，在一个月之内的时间，Linux操作系统的源码就交给Git管理了。Git迅速成为最流行的分布式版本控制工具。2008年Github网站上线，其为开源项目提供免费的代码存储，无数开源项目迁移至Github，如React，TypeScript等。

为什么Git是分布式版本控制工具？它和集中式版本控制工具SVN有什么不同呢？集中式版本控制的版本库是集中存放在中央服务器的，工作时必须首先从中央服务器获取最新版本，工作结束后再把自己的劳动成果推送给中央服务器。分布式服务器没有中央服务器，每个人电脑上都是一个完整的版本库，工作成果首先会直接存储在本地仓库，然后可以推送给远程仓库。相较集中式版本控制，分布式版本控制具有更安全，更便捷等的优点。

Git的仓库控制

工作目录：我们进行代码编辑工作的工作文件/文件夹称为工作目录。
暂存区：代码管理时首先使用git add < file >命令将代码存放到暂存区，git add .可以将工作目录所有文件都存放到暂存区。
本地仓库：然后使用git commit < file > -m 'some info here'将代码上传到本地仓库。
远程仓库：使用git push推送代码到远程仓库，它是代码托管的远程平台，用于存放最终版代码。

常用Git辅助命令

Git命令 解释
git init 初始化一个文件夹，使之被Git管理
git status 显示当前仓库的状态
git diff 查看工作区和暂存区的不同
git log 查看commit的全部记录
git log –graph 查看branch合成图
git reflog 每一次命令的记录
git rm < file > 删除暂存区文件

Git的回退操作

  • 工作区回退：当想放弃工作修改/编辑的内容时，使用git checkout -- < file >。
  • 暂存区回退：当想放弃提交到暂存区的文件时，使用git reset HEAD < file >，如果想再放弃工作区的修改/编辑，使用1的操作。
  • 本地仓库回退：当放弃本地仓库的文件时，使用git reset --(hard|mixed|soft) (HEAD\^+|HEAD~number|commit-id)。
    其中hard参数回退全部，包括HEAD、index和working tree；mixed回退部分，包括HEAD和index；soft仅回退HEAD。
    参数number和commit-id可以通过git reflog找到，commit-id不需要拼写完全，只需要开头几位字符即可。

Github和.gitignore

Github是全球最大同性交友社区，网站上有各种开源项目，供我们参考、学习和做出贡献。我们常常将公司的远程仓库设置在Github上，然后从远程仓库使用git clone url到本地进行开发。

.gitignore文件内的内容将会被忽略，不再提交，如node_modules文件夹等。

Git的分支和标签

分支管理

命令 解释
git branch 查看分支
git branch < branchname > 创建分支
git checkout < branchname > 切换到分支
git checkout -b < branchname > 创建并切换到该分支
git merge < branchname > 合并branchname分支到当前分支，此时直接将指针指向branchname分支
git merge < branchname > –no-ff 合并branchname分支到当前分支，关闭Fast-Forward模式，此时当前标签将新建一个commit后与branchname标签合并
git branch -d < branchname > 删除分支
git push origin < branchname > 推送分支到远程仓库

当临时有BUG需要处理时，可以使用git stash暂时存储当前工作状态，然后去解决BUG，解决后再使用git stash pop回到原工作状态继续工作。

标签管理

命令 解释
git tag < tagname > 在当前的HEAD状态下创建一个标签
git tag < tagname > commit-id 在指定的commit下创建标签，commit-id不需要拼写完全，只需要开头几位字符即可
git tag < tagname > -m ‘tag info here’ 用于为标签增加说明信息
git show < tagname > 用于查看标签信息
git tag 用于查看所有标签
git tag -d < tagname > 删除标签
git push origin < tagname > 推送标签到远程仓库
